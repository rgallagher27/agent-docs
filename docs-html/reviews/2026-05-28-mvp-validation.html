Bugs surfaced and fixed

First write_doc attempt failed with parent commit 409f4e7: object not found. Root cause: the MCP subprocess caches a go-git Repository handle for the lifetime of the process. When the bare clone gets new objects out-of-band (a separate agent-docs fetch CLI, or a git push into the bare), go-git's packfile index does not refresh, and the freshly-arrived parent commit appears as missing.

Fixed in commit 0e42848: gitstore.Store.refresh() re-opens the bare repo via git.PlainOpen; Commit calls it before reading the parent ref/commit. Negligible overhead (PlainOpen reads .git/config and validates layout). Test TestCommit_AfterOutOfBandFetch reproduces the dogfood scenario — an external clone branches and pushes, a fresh Store fetches, the original Store writes to the new branch — and fails without the fix, passes with it.

What's intentionally not validated yet

  • Merged-URL trunk alias. Once feat/mvp-validation merges into main, the trunk-alias URL /p/agent-docs/docs-html/reviews/2026-05-28-mvp-validation.html should serve via agent-docs serve. Step-4 smoke tests already covered branch-aware URL serving; merge needs a follow-up agent-docs fetch.
  • HTTP + MCP concurrently. Both processes were not exercised at once; should be safe given gitstore's read-concurrent design, but unverified.
  • Other harnesses (Codex, OpenCode, Cursor, Zed). Same wire protocol, expected to work; untested.
  • Reads after out-of-band changes. Same packfile-cache class as the Commit bug, but reads degrade gracefully to stale-but-consistent content. refresh() can be added to ReadBlob / ListDir / ListRefs later if it bites.

Open questions surfaced during validation

  • Author identity is process-wide (from --author-name / --author-email). The LLM cannot identify the human driving the session. Future: pull from git config user.name by default, or accept a per-call author override.
  • Branch must exist before write_doc targets it. A branch_from param or a separate create_branch tool would let the LLM drive end-to-end without dev hand-holding.
  • Manual fetch for branches created outside MCP. Could auto-fetch on write to a missing ref, or watch GitHub via webhooks (ADR-003 Q12).
  • Push failures surface as isError=true tool results — correct, but the agent can't distinguish auth vs. non-fast-forward vs. network. Structured error codes would help.

Verdict

MVP loop validated Walking-skeleton step 8 passes. With this document, all 9 tracker rows are complete. The MVP is shippable as a self-hostable, single-binary, LLM-native docs hub for solo use.

Multi-user auth, search, branch lifecycle banners, and second-harness validation are next-milestone work — out of scope for the skeleton but the foundation is in place to add them without re-architecting.
